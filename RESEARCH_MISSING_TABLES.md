## 2. Table Schemas (From Latest Migrations)

### user_reason_preferences

```sql
CREATE TABLE user_reason_preferences (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES auth.users,
  reason_type TEXT NOT NULL 
    CHECK (reason_type IN ('director', 'genre', 'actor', 'keyword', 'studio', 
                           'collection', 'decade', 'theme', 'recent')),
  success_count INTEGER DEFAULT 0,
  total_count INTEGER DEFAULT 0,
  success_rate NUMERIC(4,3) DEFAULT 0.5,
  last_updated TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE(user_id, reason_type)
);
```

**Purpose**: Tracks which recommendation reason types work best for each user.
When a suggestion is accepted/rejected, counts are updated to learn which types work best.

### user_feature_feedback

```sql
CREATE TABLE user_feature_feedback (
  id BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  feature_type TEXT NOT NULL,  -- actor, keyword, genre, subgenre, director, decade, collection, era
  feature_id BIGINT NOT NULL,  -- TMDB ID or hashed ID
  feature_name TEXT,           -- Human-readable name (nullable)
  positive_count INTEGER DEFAULT 0,
  negative_count INTEGER DEFAULT 0,
  inferred_preference NUMERIC(5,4),  -- Bayesian preference score (0-1)
  last_updated TIMESTAMPTZ DEFAULT now(),
  
  CONSTRAINT user_feature_feedback_unique UNIQUE (user_id, feature_type, feature_id)
);
```

**Purpose**: Fine-grained preference tracking for individual features (actors, keywords, genres, etc.).
Used to learn and boost/penalize suggestions containing user-preferred/disliked features.
